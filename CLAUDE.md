## Working in This Repository

There are no build, test, or lint commands — all changes are to Markdown documents only.

When editing policy or documentation files:
- Follow the existing Markdown formatting style (ATX headers, bold for key terms/dates)
- Keep the "Last updated" date current when modifying `privacy.md`
- Changes should be committed and pushed to the active development branch

## Branch Convention

Active development happens on feature branches prefixed with `claude/`. The `main` branch holds the stable, published documents.
